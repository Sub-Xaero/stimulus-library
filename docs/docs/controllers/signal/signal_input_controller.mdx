## Purpose

A controller that listens for `input` and `change` events on the `<input>`, `<select>` or `<textarea>` element the
controller is attached to, and notifies any other Signal controllers that have subscribed by referencing
the same `nameValue` as this controller, or the same name attribute of the element that this controller is attached to.

This controller works in conjunction with the other `Signal*` controllers to enable more complex interactions
based on changes to inputs and their values.

This controller will also synchronize the value of two inputs that have the same `nameValue` or the same name attribute.

## [Values](https://stimulus.hotwire.dev/reference/values)

| Value                         | Type    | Description                                                                                                                                                                                                                          | Default                                               |
|-------------------------------|---------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------|
| `name` (Optional)             | String  | The name of the input value whose value to react to. By default this is the name attribute of the input this controller is attached to. You can override the name used by providing this value.                                      | The `[name]` attribute of the controller root element |
| `debounceInterval` (Optional) | String  | The amount of time in milliseconds to [debounce](https://css-tricks.com/debouncing-throttling-explained-examples/) `input` events by. If you only want input events to trigger 1 second after the user stops typing, specify "1000". | `1000`                                                |
| `triggerChange` (Optional)    | Boolean | (Only applies when another SignalInputController emits a value change, and the current instance syncs to match) Whether to trigger a `change` event on the input element when the value changes.                                     | `false`                                               |
| `triggerInput` (Optional)     | Boolean | (Only applies when another SignalInputController emits a value change, and the current instance syncs to match) Whether to trigger an `input` event on the input element when the value changes.                                     | `false`                                               |

## Events
| Event                       | When                                         | Dispatched on               | `event.detail` |                            |
|-----------------------------|----------------------------------------------|-----------------------------|----------------|----------------------------|
| `signal:value:#{nameValue}` | When the value of the attached input changes | The controller root element | `value`        | The new value of the input |


## Side Effects
Installs `change` and `input` event handlers on the element this controller is attached to.

  <div class="row">

      <div class="card">
        <div class="card-body">
          <h5 class="card-title">Synchronized Inputs</h5>
          <div class="mb-3">
            <label class="form-label">Input 1</label>
            <input
              type="text"
              class="form-control"
              data-controller="signal-input"
              data-signal-input-name-value="sync"
              data-signal-input-debounce-interval-value="500"
            >
          </div>
          <div class="mb-3">
            <label class="form-label">Input 2 (synchronized)</label>
            <input
              type="text"
              class="form-control"
              data-controller="signal-input"
              data-signal-input-name-value="sync"
              data-signal-input-trigger-input-value="true"
            >
          </div>
        </div>
      </div>

      <div class="card mt-4">
        <div class="card-body">
          <h5 class="card-title">Debounced Search</h5>
          <div class="mb-3">
            <label class="form-label">Search (updates after 1 second of inactivity)</label>
            <input
              type="text"
              class="form-control"
              data-controller="signal-input"
              data-signal-input-name-value="search"
              data-signal-input-debounce-interval-value="1000"
            >
          </div>
          <div class="alert alert-info">
            Search value: <span id="search-value">-</span>
          </div>
        </div>
      </div>
    </div>
  </div>
</div>`,
    "/index.js": `
    import { Application } from '@hotwired/stimulus';
    import { SignalInputController } from 'stimulus-library';
    
    const app = Application.start();
    app.debug = true;
    app.register('signal-input', SignalInputController);

    // Listen for search value changes
    document.addEventListener('signal:value:search', (event) => {
      document.getElementById('search-value').textContent = event.detail.value || '-';
    });
`,
  }}
/>

The controller synchronizes input values and provides debounced input handling. In this example:

1. Two different use cases:
   - Two synchronized text inputs that update each other
   - A debounced search input that updates a display after 1 second of inactivity
2. Bootstrap styling for consistent appearance
3. Event handling to demonstrate value changes
