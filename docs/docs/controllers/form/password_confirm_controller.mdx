## Purpose

Controller to alert users when setting/changing their password, that their passwords don't match.


## [Actions](https://stimulus.hotwire.dev/reference/actions)
<NoActions/>

## [Targets](https://stimulus.hotwire.dev/reference/targets)

| Target     | Purpose                                                          | Default |
|------------|------------------------------------------------------------------|---------|
| `password` | All the `<input>`s you want to be included in the equality check | -       |

## [Classes](https://stimulus.hotwire.dev/reference/classes)

| Class              | Purpose                                                            | Default |
|--------------------|--------------------------------------------------------------------|---------|
| `error` (Optional) | The class to apply to all `password` targets when they don't match | -       |

## [Values](https://stimulus.hotwire.dev/reference/values)
<NoValues/>

## Events

| Event                       | When                      | Dispatched on          | `event.detail` |
|-----------------------------|---------------------------|------------------------|----------------|
| `password-confirm:match`    | When all inputs match     | The controller element | -              |
| `password-confirm:no-match` | When not all inputs match | The controller element | -              |

## Side Effects

The controller will add `change` event listeners to all `password` targets.



## How to Use

One, Two (or more, we don't presume) password fields, annotated as `data-password-confirm-target='password'`.

The controller will do nothing until all password target's values are filled in.

Once all passwords have a value:

- if all passwords **do not** match then the controller will add the `error` class (if specified) to all inputs, and emit an event for other controllers to hook into.
- if all passwords **match** then the controller will remove the `error` class (if specified) from all inputs, and emit an event for other controllers to hook into.

If the user then corrects an input, the process runs again - any applied `error` class will be removed, and the success event will fire.

### Example


<StimulusPlayground
  controller="PasswordConfirmController"
  html={`<style>
  .pw-error { border-color: #ef4444 !important; }
</style>
<div data-controller="password-confirm"
  data-password-confirm-error-class="pw-error"
  style="display:flex;flex-direction:column;gap:0.75rem;max-width:380px;">
  <div>
    <label style="display:block;font-size:0.9rem;color:#374151;margin-bottom:0.25rem;">Password</label>
    <input type="password" data-password-confirm-target="password"
      placeholder="Enter password"
      style="width:100%;padding:0.5rem;border:2px solid #d1d5db;border-radius:6px;transition:border-color 0.2s;">
  </div>
  <div>
    <label style="display:block;font-size:0.9rem;color:#374151;margin-bottom:0.25rem;">Confirm password</label>
    <input type="password" data-password-confirm-target="password"
      placeholder="Repeat password"
      style="width:100%;padding:0.5rem;border:2px solid #d1d5db;border-radius:6px;transition:border-color 0.2s;">
  </div>
  <div id="pw-status" style="font-size:0.85rem;color:#6b7280;">
    Enter and confirm your password above.
  </div>
</div>`}
  js={`document.querySelector('[data-controller="password-confirm"]').addEventListener('password-confirm:match', () => {
  document.getElementById('pw-status').textContent = 'Passwords match';
  document.getElementById('pw-status').style.color = '#16a34a';
});
document.querySelector('[data-controller="password-confirm"]').addEventListener('password-confirm:no-match', () => {
  document.getElementById('pw-status').textContent = 'Passwords do not match';
  document.getElementById('pw-status').style.color = '#dc2626';
});`}
  editorHeight={280}
/>
