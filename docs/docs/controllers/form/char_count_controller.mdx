## Purpose

Add a visual count of the number of characters in a textarea/string input, with an option for highlighting min/max characters.


## [Actions](https://stimulus.hotwire.dev/reference/actions)
<NoActions/>

## [Targets](https://stimulus.hotwire.dev/reference/targets)

| Target   | Purpose                                              | Default |
|----------|------------------------------------------------------|---------|
| `input`  | The `<input>` or `<textarea>` to count characters on | -       |
| `output` | The place where the characters count will be output  | -       |

## [Classes](https://stimulus.hotwire.dev/reference/classes)

| Action             | Purpose                                                                            |
|--------------------|------------------------------------------------------------------------------------|
| `error` (Optional) | The class to apply if the user does not meet the min/max characters (if specified) |

## [Values](https://stimulus.hotwire.dev/reference/values)

| Value | Type   | Description                                                                                                                   | Default |
|-------|--------|-------------------------------------------------------------------------------------------------------------------------------|---------|
| `min` | Number | The minimum number of character. This will add an `error` class (See: Classes) to the input if the min character are not met  | 0       |
| `max` | Number | The maximum number of character. This will add an `error` class (See: Classes) to the input if the max character are exceeded | 99,999  |

## Events
<NoEvents/>

## Side Effects

The controller will add an event listener for `input` to the `input` target.



## How to Use

<Sandpack
  template={"vanilla"}
  options={{
    externalResources: [
      "https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/css/bootstrap.min.css",
      "https://cdn.jsdelivr.net/npm/bootstrap@5.0.2/dist/js/bootstrap.bundle.min.js"
    ]
  }}
  customSetup={{
    dependencies: {
      "@hotwired/stimulus": "latest",
      "stimulus-library": "latest"
    }
  }}
  files={{
    "/index.html": `<div class="container mt-5">
  <div class="row">
    <div class="col-md-6">
      <div
        data-controller="char-count"
        data-char-count-min-value="10"
        data-char-count-max-value="100"
        data-char-count-error-class="is-invalid"
      >
        <label class="form-label">
          Enter text (10-100 characters)
        </label>
        <textarea
          class="form-control"
          data-char-count-target="input"
        ></textarea>
        <div class="form-text">
          Characters: <span data-char-count-target="output">0</span>
        </div>
      </div>
    </div>
    <div class="col-md-6">
      <div
        data-controller="char-count"
        data-char-count-min-value="5"
        data-char-count-max-value="50"
        data-char-count-error-class="is-invalid"
      >
        <label class="form-label">
          Enter text (5-50 characters)
        </label>
        <input
          type="text"
          class="form-control"
          data-char-count-target="input"
        />
        <div class="form-text">
          Characters: <span data-char-count-target="output">0</span>
        </div>
      </div>
    </div>
  </div>
</div>`,
    "/index.js": `
    import { Application } from '@hotwired/stimulus';
    import { CharCountController } from 'stimulus-library';
    
    const app = Application.start();
    app.debug = true;
    app.register('char-count', CharCountController);
`,
  }}
/>

The controller will count characters in text inputs and textareas, with optional min/max limits. In this example:

1. Two different inputs show different configurations:
   - Textarea with 10-100 character limits
   - Text input with 5-50 character limits
2. Both show the current character count and apply error styling when limits are exceeded
